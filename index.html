Case studies

TakeMeHome

Our client was participating in a research project on management of patient escapes from healthcare facilities. The client asked us to create a prototype that can be used at mental health facilities for testing.

We designed, created, and delivered a prototype system for the project. The system used low cost hardware to detect patient escape events and enable location tracking. The backend software provided real time notifications on a web client, as well as text (SMS) messages.

Our choice of hardware and software solutions provided many benefits over current solutions using RFID and GPS.

The project featured solutions such as:

  • Bluetooth Low Energy: Low cost and long battery life for location based solutions
  • Raspberry Pi 3: Low cost and small form factor
  • SMS: Text message notification for user alerts
  • Websockets: Realtime update of events and statistics across all servers and clients
  • Node.js: Fast and secure server providing API access

Kinect PhotoBooth

Our client was participating in a research project on adopting motion sensing technology into consumer products. The client asked us to develop a novel photobooth product by leveraging motion sensing technology

We used several unique components to design a novel photobooth system. The system is currently patent pending. The photobooth system and its provisional patent received high praises from the organizations that provided the research grant, and lead to discussions for further projects.

The project featured solutions such as:

  • Motion sensing: The hardware solution we chose provided fast and highly accurate motion sensing for users' limbs, hands, head, and torso
  • Websockets: Photos taken at the booth were delivered at real time for users to keep
  • Node.js: A novel arrangement of devices and server logic allowed users to retrieve the photos in less than a second, and share directly to various social media channels
  • Client-side webapp: Administrators had real time access to system status and statistics

VR Safari

Our client was participating in a research project on creating consumer products using virtual reality. The client asked us to create and publish two virtual reality mobile apps on the Google Play Store.

The client provided us with their app designs and graphical assets. We created the mobile applications, and found that many mobile phones used at the time had performance difficulty with virtual reality applications. We used techniques such as occlusion culling and light baking in order to dramatically increase the performance of the applications.

After extensive profiling and optimizations, the framerate of the application was nearly doubled, dramatically reducing headaches and increasing immersion.

The projecet feaured solutions such as:

  • Google Cardboard: Users with standard mobile phones could experience virtual reality at low cost
  • Profiling: Multiple devices were profiled for their CPU/GPU/Memory usage for performance analysis
  • Optimization techniques: Techniques often used in game development, such as occlusion culling, were leveraged to create a high performance application
  • Google Play Store: The applications were distributed to users around the world via the Play Store

Hybrid Mobile App

Our client was in the process of grant applications, and wanted to include a mobile app prototype in the proposal. Our client asked us to rapidly prototype both and Android app and an iOS app within 4 weeks in order to meet the grant application deadline. The client had a UI designer and illustrator on staff, who provided all assets to us.

In order to comfortably meet the deadline, we used hybrid app technology in order to develop both the Android and iOS apps simulatneously. After we recieved the client's specifications and assets, we were able to complete and test the application within a couple weeks. As we were ahead of the deadline, we suggested and implemented multiple UX improvements before delivery.

The application featured features such as:

  • Responsive layout
  • High performance application with smooth transition animations
  • API data access and submissions
  • User authentication
  • Persistent user data and settings
  • Local saves for user progress/app state
  • Accurate stopwatch and media player

Inventory Database Solution

Our client posted a request to develop a plugin for Microsoft Access that can generate PDF quotes using the data.

The client's company was just about to start a computer database, and had not picked a database solution. We sent in a proposal for a complete system including the database, tag based search, and the PDF quote generation. The client was happy with the proposal, and we had the previlege of setting up the company's database system from ground up.

The system featured a SQL database accessible by Microsoft Access, and allowed the client to create tags for each items to easily search for them when building custom quotes. The PDF quote generation system automatically inserted price, image, dimensions, and a hyperlink to access more details.

The system included features such as:

  • Simple interface that can be modified with HTML/CSS
  • Extensive item search system including text match, value range, and tags
  • PDF document generation according to client's design specifications
  • An "Open All" button in the PDF that opens multiple product listings with one click
  • Detailed installation, usage, and troubleshooting guide
